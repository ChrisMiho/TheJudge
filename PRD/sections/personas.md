# personas.md

## Primary Persona

### PERSONA-001
- Name: Casual to Semi-Competitive MTG Player
- Role: Active player at a table trying to understand a stack interaction
- Environment: Usually mobile phone during live gameplay
- Primary Need: Fast clarification without leaving the moment of play
- Pain Points:
  - stack interactions are unclear
  - manual lookup is slow
  - table disagreement is common
  - generic AI tools lack exact stack context

## Audience Constraints
- users are likely multitasking during a game
- users need low-friction input
- users may not enter detailed structured data
- users need confidence, but not false certainty

## Design Implications
- mobile-first UX
- narrow context
- minimal required fields
- fast search and add loop
- clear stack ordering
- plain, usable response output
